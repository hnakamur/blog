はじめに

最近買ったデスクトップPCのWindows 11 HomeのWSL2にLAN内のリモートマシンからsshしたいと思って調べたら、先人の記事でサクッとできたのでメモ。

WindowsのOpenSSHサーバに接続した後、localhostにsshすることでWSLのOpenSSHサーバに接続するという方式です。

多段sshの設定

クライアント側の ~/.ssh/config に以下のような設定をします。通常の多段sshの設定ですが、WindowsホストからlocalhostでWSLに接続できるのがミソです。

Host ggear
  Hostname localhost
  User hnakamur
  ForwardAgent yes
  ProxyCommand ssh -W %h:%p ggearwin

Host ggearwin
  Hostname 192.168.2.11
  User hnaka
  ForwardAgent yes

ServerAliveInterval 15

WSLの設定

/etc/wsl.conf を作成し systemd=true を追加

[boot]
systemd=true

openssh-serverをインストールして自動起動を有効化

sudo apt-get install -y openssh-server

sudo systemctl enable ssh

~/.ssh/authorized_keys に公開鍵を設置します。

Windowsの設定

C:\ProgramData\ssh\sshd_config の最後の2行をコメントアウト

行頭に # を入れてコメントアウトします。VS Codeで保存しようとすると、右下にポップアップが出るので[Retry as Admin]ボタンを押せば管理者権限で保存してくれました。

#Match Group administrators
#       AuthorizedKeysFile __PROGRAMDATA__/ssh/administrators_authorized_keys

この設定があると $HOME/.ssh/authorized_keys に公開鍵を置いても見てくれないとのことです（ここの $HOME はWindows側のホームディレクトリのこと））。 接続できるようになった後、一度戻してsshdを再起動してみたのですが接続できなくなりました。

$HOME/.ssh/authorized_keys に公開鍵を設置します。

OpenSSHサーバの起動と自動起動の有効化

PowerShellを管理者権限で起動して、以下のコマンドを入力。

Start-Service sshd

Set-Service -Name sshd -StartupType 'Automatic'

ファイアウォールでOpenSSHへの接続を許可

PowerShellを管理者権限で起動して、以下のコマンドを入力。

事前の設定確認（未設定時は出力は空）。

Get-NetFirewallRule -Name *ssh*

ポートを開ける。

New-NetFirewallRule -Name sshd -DisplayName 'OpenSSH Server (sshd)' -Enabled True -Direction Inbound -Protocol TCP -Action Allow -LocalPort 22

事後の設定確認。

Get-NetFirewallRule -Name *ssh*

Wake on LAN の設定

私のPCは有線のネットワークアダプタで接続しています。

  1. Win + X で開くメニューからデバイスマネージャを起動。
  2. ネットワークアダプターを開いて対象のネットワークアダプタを選択してダブルクリック。
  3. 「電源の管理」タブに切り替えて以下の2つにチェックがついた状態にする。
    • 「このデバイスで、コンピュータのスタンバイ状態を解除できるようにする」
    • 「Magic Packetでのみ、コンピュータのスタンバイ状態を解除できるようにする」
